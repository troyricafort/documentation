# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

This is a **docs-as-code** repository. Markdown files under `docs/` are the source of truth. On every PR merge to `main`, a GitHub Actions workflow syncs changed files to Google Drive as native Google Docs, mirroring the repo folder structure.

## Running the scripts locally

Both scripts require two environment variables:

```bash
export GOOGLE_SERVICE_ACCOUNT_KEY=/path/to/sa-key.json
export GDRIVE_ROOT_FOLDER_ID=your_drive_folder_id
```

Install dependencies:

```bash
pip install -r requirements.txt
```

**Bootstrap** (first time only — seeds all of `docs/` into Drive):

```bash
python scripts/bootstrap.py
```

**Delta sync** (sync specific files manually):

```bash
python scripts/sync.py docs/service-records/overview.md
# or via env var:
CHANGED_FILES="docs/service-records/overview.md" python scripts/sync.py
# with deletions:
DELETED_FILES="docs/service-records/old-file.md" python scripts/sync.py
```

## Architecture

### Data flow

```
docs/*.md  →  PR merged to main  →  GitHub Actions  →  sync.py  →  Google Drive
```

The workflow (`sync-to-gdrive.yml`) fires only on merged PRs, fetches the list of changed `.md` files from the PR via the GitHub API, and passes them to `sync.py` via env vars.

### gdoc-map.json

The critical state file. Maps every `docs/`-relative file path to its Google Doc ID:

```json
{
  "docs/service-records/overview.md": "1abc...GDocId"
}
```

- `bootstrap.py` writes this from scratch on first run
- `sync.py` reads and updates it on every PR merge
- The workflow commits any changes back to `main` automatically
- **Do not edit this file manually**

### sync.py logic

- File **added or modified** → upsert: update existing GDoc if ID is in map, otherwise create a new one and add to map
- File **deleted** → archive: trash the GDoc and remove entry from map
- Folder structure in Drive is created on demand via `ensure_folder()`, which checks before creating to avoid duplicates

### bootstrap.py vs sync.py

`bootstrap.py` does a full walk of `docs/` and overwrites `gdoc-map.json` entirely. Only run it once (or when recovering from a corrupted map). `sync.py` is the routine path and only touches files that changed in the PR.

## Doc contribution workflow

1. Create a branch
2. Add or edit `.md` files under `docs/`
3. Open a PR — Claude Code can help draft content from specs or diffs
4. Merge to `main` — Drive is updated automatically
